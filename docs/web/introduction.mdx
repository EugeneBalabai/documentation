Tolgee SDK provides functionality of i18n library and helps you with formatting, interpolation etc.
It also serves as a bridge to Tolgee Platform for [in-context translating](/features/dev-tools#in-context).

Basic initialization:

```ts
import { Tolgee } from '@tolgee/web';

const tolgee = Tolgee({
  language: 'en',
  staticData: {
    en: {
      test: 'Hello world!',
    },
    es: {
      test: '¡Hola Mundo!',
    },
  },
});
tolgee.run();

console.log(tolgee.t('test'));
// Hello world!
```

We can also switch languages:

```ts
tolgee.changeLanguage('es');

console.log(tolgee.t('test'));
// ¡Hola Mundo!
```

> Read more about [`staticData`](./core-package/options#staticdata).

## Using plugins

Tolgee is highly extensible and we offer variety of plugins directly included in SDK.
Plugins can add support for:

- Formatting
- Backends (fetching translations)
- Language detectors
- Language storage
- In context translating
- [... and more](./core-package/plugin-api)

This is how you can add `FormatSimple` plugin:

```ts
import { FormatSimple } from '@tolgee/web';

const tolgee = Tolgee()
  .use(FormatSimple())
  .init({
    ...
  });
```

> The [`init`](./core-package/tolgee#init) method is equivalent to regular initialization,
> but is more elegant when using plugins.

## Using in-context translating

To enbale in-context translating, we need to use `DevTools` plugin and specify Tolgee credentials.

```ts
import { Tolgee, DevTools } from '@tolgee/web';

const tolgee = Tolgee()
  .use(DevTools())
  .init({
    language: 'en',
    staticData: {
      ...
    },
    apiUrl: 'https://app.tolgee.io',
    apiKey: '<your project API key>',
  });
```

> Api key can be [obtained in Tolgee Platform](../platform/api-keys-and-pat-tokens).

If we want to try in-context, we need to insert the translation to the dom.

```ts
tolgee.run();

myHtmlElement.textContent = tolgee.t('test');
tolgee.on('keyUpdate', () => {
  myHtmlElement.textContent = tolgee.t('test');
});
```

For in-context translating is important, that translation can change any time, so we need to
listen for these changes and update html.

This exmple might look a bit ugly, but that's only because it's in vanilla js. If you use any of our
framework integrations, you'll see, it's much more elegant.

> How does Tolgee know where are the translations on the page? Check [Wrapping and Observing](./wrapping)
